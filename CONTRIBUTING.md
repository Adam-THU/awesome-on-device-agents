# Contributing

Contributions are welcome.

## What to Add

This repository prioritizes recent work from 2024 onward. Older work may be included only when it is foundational for on-device agents, mobile GUI automation, or efficient edge inference.

Good additions should be related to at least one of:

- mobile GUI agents;
- mobile app automation;
- multimodal mobile agents;
- personal mobile agents;
- device-cloud collaborative agents;
- on-device LLM / VLM / SLM inference;
- edge inference optimization;
- mobile agent benchmarks or datasets;
- open-source systems for local inference, mobile AI engines, phone agents, or device-cloud handoff.

## Selection Criteria

Please prefer papers or systems that satisfy at least one of:

- Strong relevance to on-device agents, mobile GUI agents, or Android task automation.
- Strong relevance to efficient on-device inference, edge LLM serving, or collaborative edge inference.
- Published in a major venue, widely cited, or already influential in the community.
- Very recent and directly aligned with practical on-device agent deployment.
- Useful benchmark, dataset, runtime, or open-source system for this research area.

Avoid adding:

- General web agents with no mobile, device, or edge angle.
- Cloud-only agent systems with no device-side deployment relevance.
- Pure model capability papers without a clear link to agents, mobile systems, or edge inference.
- Duplicate entries or papers with unverified metadata.

## Metadata Rules

- Use `YYYY.M` for dates when month is known.
- Use the publication, acceptance, or conference date when available; otherwise use the arXiv release date.
- Use short venue names such as `MobiCom`, `MobiSys`, `TMC`, `INFOCOM`, `arXiv`.
- If a paper has both an official publication page and an arXiv page, either is acceptable as long as the link is correct.

## Style

- Prefer structured notes over long summaries.
- Mark uncertain metadata as `TODO` instead of guessing.
- Include paper, code, project, dataset, or benchmark links when available.
- Keep entries focused on why the work matters for mobile agents or edge inference.
